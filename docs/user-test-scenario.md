# user-test-scenario.md — ユーザーテスト用デモシナリオ

Sprint 10「ユーザーテスト準備」向け。実際の経営者・士業にSUNBOOを操作してもらい、
正式リリース前の課題を洗い出すための操作手順。所要時間の目安は10〜15分。

対応エリアは現状**福岡県・東京都渋谷区のみ**（β版）。テスト時はテスターに事前にその旨を伝えること
（[tester-message.md](tester-message.md)参照）。

操作後は[user-test-feedback.md](user-test-feedback.md)に沿ってフィードバックをもらう。

## シナリオ手順

### 1. 初めてSUNBOOを開く
- トップページ（`/`）にアクセスしてもらう
- 何も説明せず、**第一印象だけで**「これは何をするサービスか」を口頭で説明してもらう
  （フィードバックシートの「最初に何をするサービスか分かったか」に対応）

### 2. 会社情報を入力する
- トップページの「はじめる」から`/start`へ進んでもらう
- 都道府県・市区町村・従業員の有無・決算月・法人の種類を入力してもらう
- 迷った箇所・入力しづらかった項目があれば都度メモする

### 3. 診断結果を見る
- `/result`の管轄機関一覧・必要手続き一覧をひと通り眺めてもらう
- 情報量やレイアウトについて感想をもらう

### 4. AI参謀カードを見る
- ダッシュボード上部の「AI参謀」カードに気づくか確認する
- 「最重要アクション」「次に来る予定」「注意すべきリスク」「優先度」の4セクションが
  それぞれ何を意味しているか、説明を読まずに理解できるか確認する

### 5. 通知カードを見る
- AI参謀カードの上に表示される「通知」カードに気づくか確認する
  （該当する手続きが無い場合は表示されないため、事前に期限が近いデモデータで確認するか、
  表示されない状態そのものも「情報が無ければ何も出ない」という挙動として説明する）
- AI参謀カードとの違い（通知＝期限の知らせ、AI参謀＝判断）が伝わるか確認する

### 6. 手続き詳細を見る
- 必要手続き一覧から1件選び、「詳細を見る」を開いてもらう
- 必要書類・提出方法・注意書き（caution_note）が実務で役立つ情報か確認する

### 7. 手続きのステータスを更新する
- 任意の手続きのチェックボタンを押し、未着手→進行中→完了と変化させてもらう
- 完了にした手続きが、AI参謀カード（最重要アクション・優先度・リスク）や通知カードから
  リアルタイムに外れることに気づくか確認する

### 8. イベントを登録する
- `/events`から「会社設立」など任意のイベントを選び、登録してもらう
- 登録後に必要手続きが自動生成され、AI参謀カード・通知カードにも反映されることを確認する

## テスター向けの前提条件

- Supabaseの環境変数が設定済みの環境（本番 or ローカル）で実施すること
- 初回アクセス時は`localStorage`が空の状態から始めること（別の会社データが残っていると
  診断結果が意図と異なる場合がある。ブラウザのシークレットウィンドウ推奨）
- 対応エリア外（例: 東京都新宿区）を選んだ場合は「現在未対応です」と表示される。
  これは不具合ではなく仕様であることをテスターに伝えておく
