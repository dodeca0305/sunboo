# SUBMISSION_DIRECTORY_ROADMAP.md — Data Gap Report（Phase5-4）

**作成日**: 2026-07-17
**目的**: 全国自治体のうち、Submission Directoryへ追加すべき自治体を優先順位付きで整理する。**本書はデータ追加の実施計画ではなく、ギャップの棚卸しである。** 実際の調査・投入手順は[MUNICIPAL_DISCOVERY_CHECKLIST.md](MUNICIPAL_DISCOVERY_CHECKLIST.md)（SOP、レビュー待ち）に既に定義済みのため、本書はそれを参照し重複させない。
**優先順位**: ユーザー指示により「①政令指定都市 → ②中核市 → ③東京都特別区 → ④その他」の順で整理する。

---

## 0. 前提: 「地理マスタが存在する」と「窓口データがある」は別軸

[RESOLVER_COVERAGE.md](RESOLVER_COVERAGE.md)で確認した通り、`municipalities`（地理マスタ）にcanonical
6桁コードが存在することと、`submission_offices`/`submission_jurisdictions`（窓口データ）が
投入されていることは独立している。本書のギャップ表はこの2つを分けて記載する。

## 2. 優先順位② 中核市

**本節は「要検証」であることを明示する。** 中核市の指定基準・指定状況は地方自治法改正や個別市の指定により変動するため、本セッションでは総務省等の一次情報源に基づく最新の正確な市数・市名一覧を検証していない（DBアクセスがanon keyのみで確認手段が無く、Web検索によるファクトチェックもこのタスクの範囲外としている）。**データ調査に着手する前に、まず総務省の最新の中核市一覧を一次情報源で確認するステップを独立したタスクとして設けること。**

現時点で言えること:

- 中核市は人口20万人以上の要件で指定される市であり、2020年代時点で60市前後が指定されている認識だが、**本書では正確な件数・市名一覧を断定しない**（[MUNICIPAL_DISCOVERY_CHECKLIST.md](MUNICIPAL_DISCOVERY_CHECKLIST.md) 1-1節「Unknown is better than Wrong」の対象は窓口データだけでなく、この種の制度上の事実確認にも同様に適用すべきと判断する）
- 中核市は政令指定都市と異なり「区」を持たない単一の`municipality_code`であることが多い（要確認）ため、[MUNICIPAL_DISCOVERY_CHECKLIST.md](MUNICIPAL_DISCOVERY_CHECKLIST.md) 4節「政令指定都市の追加確認事項」は不要になる可能性が高いが、これも個別市ごとに確認する
- 地理マスタ（`municipalities`）への中核市データ投入状況も本書では未確認（`migration_designated_cities_geography.sql`は政令指定都市のみが対象と明記されており、中核市は対象外と読める）

**推奨する次アクション**: 中核市着手前に、(a) 総務省公式サイト等の一次情報源で現行の中核市一覧を確認し、(b) 地理マスタへの投入要否を確認する、という2ステップを専用の調査タスクとして切り出す。本書では優先順位の「位置づけ」のみを示す。

## 3. 優先順位③ 東京都特別区（23区）

| 区分 | 状態 |
|---|---|
| 地理マスタ | **渋谷区（`131130`）のみ投入済み。残り22区は`municipalities`に一切存在しない**（確認済み: `migration_organizations.sql`・`migration_designated_cities_geography.sql`双方に渋谷区以外の東京23区コードは1件も無い） |
| 窓口データ（新Resolver） | 渋谷区分もmigrationの区分としては「旧スキーマ」（`organizations`/`jurisdictions`）に存在するのみで、`submission_offices`（新スキーマ）には移植されていない（Phase3Aの移植対象は福岡県のみ、渋谷区は対象外と明記） |
| 窓口データ（旧Resolver） | 渋谷区のみ7カテゴリ完備。残り22区は0件 |

特別区は23区で確定した制度（政令指定都市・中核市のような指定基準の変動が無い）ため、**本書内で最も事実確認コストが低い優先順位**。ただし着手にあたっては以下の東京都特有の事情に注意する必要がある（[RESOLVER_COVERAGE.md](RESOLVER_COVERAGE.md)にも既存の暗示あり、Procedure Master側のnotes欄）。

- `MUNICIPAL_RESIDENT_TAX_RETURN`（法人市民税申告）のProcedure Master notesに「東京23区は都税事務所への申告に一本化される等、自治体により扱いが異なる場合があります」と明記されている。特別区は市町村税を都が徴収する特例（都税事務所が市町村民税相当分も扱う）があるため、他の政令指定都市・一般市と同じ「区ごとのmunicipal_tax窓口」という前提が成立しない可能性が高い。**着手前にこの制度差を一次情報で確認するステップが必須**（[MUNICIPAL_DISCOVERY_CHECKLIST.md](MUNICIPAL_DISCOVERY_CHECKLIST.md) 1-3節の「推測による登録の禁止」に該当するリスクが高い領域）
- 地理マスタ自体（22区分の`municipality_code`投入）から着手する必要があり、政令指定都市（マスタ済み）より1ステップ多い

## 4. 優先順位④ その他（一般市区町村）

- 福岡県内の残り58判定単位（一般市・町村、地理マスタは既存）が、優先順位としては地理的近接性・スキーマ実績（Phase3B/3Cで既に運用実績あり）の観点から最も着手しやすい「その他」
- 全国の一般市区町村（政令指定都市・中核市・特別区を除く、約1,700団体規模）は[MUNICIPAL_DISCOVERY_CHECKLIST.md](MUNICIPAL_DISCOVERY_CHECKLIST.md) 11節が既に「本格的な全国展開の対象」と位置づけている
- 全国展開の体制（人手維持か半自動化か）は[NATIONAL_SUBMISSION_DIRECTORY.md](NATIONAL_SUBMISSION_DIRECTORY.md) D12で「Phase2実績を待って再評価、保留」のまま。本書はこの判断を先取りしない

`tax_office`/`legal_affairs_bureau`/`pension_office`/`labor_standards`/`hello_work`/`prefectural_tax`の6カテゴリは、福岡県内では既に72市区町村（＝県内一般市区町村も含め）投入済みだが、**福岡県以外の46都道府県ではこの6カテゴリも0件**である点に注意（[RESOLVER_COVERAGE.md](RESOLVER_COVERAGE.md) 1節）。「その他」を検討する際は、municipal_tax/municipal_asset_taxだけでなくこの6カテゴリの全国展開も別軸のギャップとして残っていることを見落とさないこと。

## 5. 優先順位まとめ（次に着手すべき1件を選ぶなら）

| 順位 | 対象 | 理由 |
|---|---|---|
| 1 | 北九州市・資産税（`municipal_asset_tax`） | 地理マスタ・市民税側の実装済み。資産税担当部署の一次確認のみで完結する最小タスク |
| 2 | 政令指定都市の残り17市（地理マスタ済み） | SOP（[MUNICIPAL_DISCOVERY_CHECKLIST.md](MUNICIPAL_DISCOVERY_CHECKLIST.md)）が確立済みで、Step 0を省略できる |
| 3 | 中核市の一覧確認（データ投入ではなく事実確認タスク） | 制度理解が不確実なまま着手するとVISION.mdの「推測による登録の禁止」に抵触するリスクがあるため、着手前の調査を独立タスク化する |
| 4 | 東京都特別区（地理マスタ投入＋都税事務所一本化の制度確認） | 22区分の地理マスタが未投入・かつ制度差の確認が必要で、政令指定都市より着手コストが高い |
| 5 | 一般市区町村の全国展開 | 体制判断が保留中（D12）のため、体制判断が下るまでは福岡県内の残り58判定単位に限定するのが安全 |
