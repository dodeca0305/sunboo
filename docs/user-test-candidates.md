# user-test-candidates.md — テスター候補リスト（テンプレート）

Sprint 12「βテスト実施」向け。**実在の候補者名はこのファイルに含まれていません**
（社内の人脈情報のため、AIが名前を創作・推測することはできません）。以下は記入用テンプレートです。
各カテゴリの調達ヒントを参考に、実際の候補者を埋めてください。

## 目標人数

| 属性 | 目標人数 | 優先度 | 理由 |
|---|---|---|---|
| 起業予定者 | 2〜3名 | 高 | 「初めて触る」第一印象を最も汚れなく評価できる層 |
| 起業1年以内の経営者 | 2〜3名 | 高 | 実際に手続きに追われた経験があり、実務との答え合わせができる |
| 税理士 | 1〜2名 | 中 | 情報の正確性・顧問先への推薦可能性を評価できる |
| 社労士 | 1〜2名 | 中 | 労務・社保系の手続き（通知・リスク文言）の実務的な妥当性を評価できる |
| 行政書士 | 1〜2名 | 中 | 法人設立登記・許認可系の情報の正確性を評価できる |

合計 8〜13名程度を目安にする（最初のβテストとしては多すぎず、傾向が見える最小限の人数）。

## カテゴリ別・調達のヒント

- **起業予定者**: 商工会議所・自治体の創業支援窓口、創業スクール/セミナーの参加者、SNSで「起業準備中」を発信している人
- **起業1年以内の経営者**: 上記と同じ経路に加え、法人設立freee/マネーフォワード等のコミュニティ、地域の経営者交流会
- **税理士・社労士・行政書士**: 既に顧問関係にある士業、知人経由の紹介、各士業会（税理士会・社労士会・行政書士会）の若手向け勉強会
- 士業は「自分が使う」というより「顧問先に薦められるか」という視点で評価してもらうことが多いため、
  依頼時に[tester-message.md](tester-message.md)の士業向け補足を添えること
