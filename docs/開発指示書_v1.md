## 1. プロダクト要件

### 何を作るか

**SUNBOO経営ナビ** — 法人の所在地・従業員有無・決算月・業種を入力すると、
提出すべき書類・提出先・期限・公式リンクを一覧表示する Webアプリ。

### 誰のため

- 法人を設立したばかりの経営者
- 顧問税理士・社労士がいない中小企業
- 毎年何を出すか把握していない事業主

### やらないこと（MVP）

- ユーザー登録・ログイン（MVP は匿名）
- AI チャット（後フェーズ）
- 書類ダウンロード・電子申請連携
- モバイルアプリ
- 決済

## 2. MVP スコープ

### 入力（MVP 固定パラメータ）

| 項目 | MVP の対象 |
|------|-----------|
| 会社所在地 | 東京都 渋谷区 |
| 従業員 | あり（人数は問わない） |

| 業種 | 指定なし（全業種共通の手続きのみ） |

### 出力：管轄機関（6機関）

| # | 手続き名 | 種別 | 管轄機関種別 | タイミング |

### 3-2. `/start` 会社情報入力

```
┌────────────────────────────────────────┐
│  ← SUNBOO経営ナビ                      │
├────────────────────────────────────────┤
│  会社情報を入力してください             │
│                                        │
│  ① 会社の所在地                        │
│  都道府県 [東京都         ▼]           │
│  市区町村 [渋谷区         ▼]           │
│                                        │
│  ② 従業員はいますか？                  │
│  [  あり  ] [  なし  ]                 │
│                                        │
│  ③ 決算月                              │
│  [ 3月           ▼]                   │
│                                        │
│  (将来) ④ 業種                         │
│                                        │
│       [診断結果を見る →]               │
└────────────────────────────────────────┘
```

**フォーム仕様:**
- 都道府県セレクト → 選択後に市区町村セレクトを表示（連動）
- 従業員: トグルボタン（あり/なし）
- 決算月: 1〜12月のセレクト
- バリデーション: 全項目必須
- 送信: `/result?pref=13&muni=13113&emp=true&fm=3` にリダイレクト

### 3-3. `/result` 診断結果

```
┌────────────────────────────────────────┐
│  ← 条件を変更する                      │
│                                        │
│  【診断結果】                          │
│  東京都渋谷区 / 従業員あり / 3月決算   │
│                                        │
│  ─── 管轄機関（6件）────────────────  │
│  ┌──────────────────┐                 │
│  │ 🏛 渋谷税務署      │                 │
│  │ 渋谷区神山町...    │ [地図] [サイト]  │
│  └──────────────────┘                 │
│  （以下5機関）                         │
│                                        │
│  ─── 必要手続き（10件）──────────────  │
│  ┌──────────────────────────────────┐ │
│  │ [登録] 法人設立届出書              │ │
│  │ 提出先: 渋谷税務署                 │ │
│  │ 期限:  設立から2ヶ月以内           │ │
│  │                     [公式サイト →] │ │
│  └──────────────────────────────────┘ │
│  （以下9手続き）                       │
└────────────────────────────────────────┘
```

**表示ロジック:**
1. URL クエリパラメータを読み取る
2. `municipalities` テーブルから municipality を特定
3. その municipality に紐づく `jurisdiction_offices` を取得
4. `procedures` を `requires_employees` と `applicable_industries` でフィルタ
5. 各 procedure に対し、`office_type` で管轄機関を特定
6. 期限を計算して表示

## 4. DB 設計（完全スキーマ）

```sql
-- ============================================================
-- SUNBOO経営ナビ Supabase スキーマ v1.0
-- ============================================================

-- 都道府県
CREATE TABLE prefectures (
  id         SERIAL      PRIMARY KEY,
  code       CHAR(2)     NOT NULL UNIQUE,   -- JIS都道府県コード '13'
  name       VARCHAR(20) NOT NULL           -- '東京都'
);

-- 市区町村
CREATE TABLE municipalities (
  id            SERIAL      PRIMARY KEY,
  prefecture_id INTEGER     NOT NULL REFERENCES prefectures(id),
  code          CHAR(5)     NOT NULL UNIQUE, -- JIS市区町村コード '13113'（渋谷区）
  name          VARCHAR(50) NOT NULL,        -- '渋谷区'
  UNIQUE(prefecture_id, code)
);

-- 管轄機関
-- ひとつの市区町村に対して複数の管轄機関が紐づく
CREATE TABLE jurisdiction_offices (
  id              SERIAL       PRIMARY KEY,
  municipality_id INTEGER      NOT NULL REFERENCES municipalities(id),
  office_type     VARCHAR(30)  NOT NULL,
    -- 'tax_office'      : 税務署
    -- 'prefectural_tax' : 都道府県税事務所
    -- 'municipal_tax'   : 市区町村（市民税）
    -- 'pension_office'  : 年金事務所
    -- 'labor_standards' : 労働基準監督署
    -- 'hello_work'      : ハローワーク（公共職業安定所）
    -- 'labor_bureau'    : 都道府県労働局
  name            VARCHAR(100) NOT NULL,   -- '渋谷税務署'
  address         TEXT,
  phone           VARCHAR(20),
  website_url     TEXT,
  map_url         TEXT,
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- 手続きマスタ
CREATE TABLE procedures (
  id                   SERIAL       PRIMARY KEY,
  code                 VARCHAR(50)  NOT NULL UNIQUE, -- 'CORP_TAX_ESTABLISH'
  name                 VARCHAR(100) NOT NULL,
  description          TEXT,
  category             VARCHAR(20)  NOT NULL,
    -- 'tax'          : 税務
    -- 'labor'        : 労務
    -- 'insurance'    : 社会保険
    -- 'registration' : 登記・届出
  requires_employees   BOOLEAN      NOT NULL DEFAULT FALSE,
  applicable_industries TEXT[],  -- NULL = 全業種
  office_type          VARCHAR(30)  NOT NULL, -- jurisdiction_offices.office_type と対応
  frequency            VARCHAR(20)  NOT NULL,
    -- 'one_time'   : 設立時など一度のみ
    -- 'annual'     : 年1回
    -- 'semi_annual': 年2回
    -- 'monthly'    : 毎月
    -- 'event'      : イベント発生時
  timing_label         VARCHAR(100) NOT NULL, -- 人間向け期限説明 '設立から2ヶ月以内'
  timing_type          VARCHAR(30)  NOT NULL,
    -- 'at_establishment' : 設立時
    -- 'hiring_event'     : 採用時
    -- 'fiscal_offset'    : 決算月からNヶ月後
    -- 'fixed_date'       : 毎年固定日
    -- 'period'           : 毎年○月○日〜○月○日
    -- 'monthly_10th'     : 毎月10日
  timing_data          JSONB,
    -- fiscal_offset: { "months": 2 }
    -- fixed_date:    { "month": 1, "day": 31 }
    -- period:        { "startMonth": 6, "startDay": 1, "endMonth": 7, "endDay": 10 }
  priority             INTEGER      NOT NULL DEFAULT 0, -- 表示順
  is_active            BOOLEAN      NOT NULL DEFAULT TRUE,
  created_at           TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

-- 手続き必要書類
CREATE TABLE procedure_documents (
  id           SERIAL       PRIMARY KEY,
  procedure_id INTEGER      NOT NULL REFERENCES procedures(id) ON DELETE CASCADE,
  name         VARCHAR(200) NOT NULL, -- '法人設立届出書'
  form_number  VARCHAR(50),           -- '別紙2（国税庁）'
  is_required  BOOLEAN      NOT NULL DEFAULT TRUE,
  notes        TEXT,
  sort_order   INTEGER      NOT NULL DEFAULT 0
);

-- 公式リンク
CREATE TABLE official_links (
  id           SERIAL  PRIMARY KEY,
  procedure_id INTEGER REFERENCES procedures(id) ON DELETE CASCADE,
  office_id    INTEGER REFERENCES jurisdiction_offices(id) ON DELETE CASCADE,
  label        VARCHAR(200) NOT NULL, -- 'e-Taxで提出する'
  url          TEXT         NOT NULL,
  sort_order   INTEGER NOT NULL DEFAULT 0,
  created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  CHECK (procedure_id IS NOT NULL OR office_id IS NOT NULL)
);

-- 診断入力（匿名セッション保存）
CREATE TABLE companies (
  id              UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  session_id      VARCHAR(100),
  prefecture_id   INTEGER     REFERENCES prefectures(id),
  municipality_id INTEGER     REFERENCES municipalities(id),
  has_employees   BOOLEAN     NOT NULL DEFAULT FALSE,
  employee_count  INTEGER,
  fiscal_month    INTEGER     NOT NULL CHECK (fiscal_month BETWEEN 1 AND 12),
  industry_code   VARCHAR(20),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- インデックス
CREATE INDEX idx_municipalities_pref    ON municipalities(prefecture_id);
CREATE INDEX idx_offices_municipality   ON jurisdiction_offices(municipality_id);
CREATE INDEX idx_offices_type           ON jurisdiction_offices(office_type);
CREATE INDEX idx_procedures_category    ON procedures(category);
CREATE INDEX idx_procedures_active      ON procedures(is_active);
CREATE INDEX idx_links_procedure        ON official_links(procedure_id);
CREATE INDEX idx_links_office           ON official_links(office_id);
```

```typescript
// src/lib/diagnosis.ts

import { createClient } from '@supabase/supabase-js';

export type DiagnosisInput = {
  prefectureCode: string;    // '13'
  municipalityCode: string;  // '13113'
  hasEmployees: boolean;
  fiscalMonth: number;       // 1〜12
  industryCode?: string;     // 将来用
};

export type JurisdictionOffice = {
  id: number;
  office_type: string;
  name: string;
  address: string | null;
  phone: string | null;
  website_url: string | null;
  map_url: string | null;
};

export type ProcedureResult = {
  id: number;
  code: string;
  name: string;
  description: string;
  category: string;
  office_type: string;
  frequency: string;
  timing_label: string;
  timing_type: string;
  timing_data: Record<string, unknown> | null;
  next_deadline: string | null;      // 計算済み期限
  office: JurisdictionOffice | null; // 紐づく管轄機関
  official_links: { label: string; url: string }[];
};

export type DiagnosisResult = {
  offices: JurisdictionOffice[];
  procedures: ProcedureResult[];
};

// 期限計算
function calculateNextDeadline(
  timingType: string,
  timingData: Record<string, unknown> | null,
  fiscalMonth: number,
): string | null {
  const today = new Date();
  const year = today.getFullYear();

  switch (timingType) {
    case 'at_establishment':
    case 'hiring_event':
      return null; // イベント起算は計算不可

    case 'fiscal_offset': {
      const months = (timingData?.months as number) ?? 2;
      const rawMonth = fiscalMonth + months;
      const monthIndex = ((rawMonth - 1) % 12);
      const deadlineMonth = monthIndex + 1;
      let deadlineYear = year;
      const d = new Date(year, monthIndex + 1, 0);
      if (d < today) {
        deadlineYear = year + 1;
      }
      const lastDay = new Date(deadlineYear, monthIndex + 1, 0).getDate();
      return `${deadlineYear}年${deadlineMonth}月${lastDay}日`;
    }

    case 'fixed_date': {
      const m = timingData?.month as number;
      const day = timingData?.day as number;
      let d = new Date(year, m - 1, day);
      if (d < today) d = new Date(year + 1, m - 1, day);
      return `${d.getFullYear()}年${m}月${day}日`;
    }

    case 'period': {
      const sm = timingData?.startMonth as number;
      const sd = timingData?.startDay as number;
      const em = timingData?.endMonth as number;
      const ed = timingData?.endDay as number;
      let endDate = new Date(year, em - 1, ed);
      const targetYear = endDate < today ? year + 1 : year;
      return `${targetYear}年${sm}月${sd}日〜${em}月${ed}日`;
    }

    case 'monthly_10th': {
      const nextM = today.getMonth() + 2;
      const m = nextM > 12 ? 1 : nextM;
      const y = nextM > 12 ? year + 1 : year;
      return `${y}年${m}月10日`;
    }

    default:
      return null;
  }
}

// メイン診断関数
export async function runDiagnosis(
  supabase: ReturnType<typeof createClient>,
  input: DiagnosisInput,
): Promise<DiagnosisResult> {
  // 1. municipality を特定
  const { data: muni } = await supabase
    .from('municipalities')
    .select('id')
    .eq('code', input.municipalityCode)
    .single();

  if (!muni) return { offices: [], procedures: [] };

  // 2. 管轄機関を取得
  const { data: offices } = await supabase
    .from('jurisdiction_offices')
    .select('*')
    .eq('municipality_id', muni.id)
    .order('id');

  // 3. 手続きを取得・フィルタ
  let query = supabase
    .from('procedures')
    .select(`
      *,
      official_links (label, url)
    `)
    .eq('is_active', true)
    .order('priority');

  if (!input.hasEmployees) {
    query = query.eq('requires_employees', false);
  }

  // 将来: 業種フィルタ
  // if (input.industryCode) {
  //   query = query.or(`applicable_industries.cs.{${input.industryCode}},applicable_industries.is.null`);
  // }

  const { data: procs } = await query;

  // 4. 各手続きに管轄機関を紐づけ・期限計算
  const officeMap = new Map(offices?.map((o: JurisdictionOffice) => [o.office_type, o]));

  const procedures: ProcedureResult[] = (procs ?? []).map((p: Record<string, unknown>) => ({
    ...p,
    next_deadline: calculateNextDeadline(
      p.timing_type as string,
      p.timing_data as Record<string, unknown> | null,
      input.fiscalMonth,
    ),
    office: officeMap.get(p.office_type as string) ?? null,
    official_links: (p.official_links as { label: string; url: string }[]) ?? [],
  }));

  return {
    offices: offices ?? [],
    procedures,
  };
}

## 診断ロジック

ルールベース（AI なし）。
`所在地 × 従業員有無 × 決算月` から DB を参照して結果を生成。

将来: `業種` によるフィルタ追加 → その後で AI 自然文検索を追加。

## DB 構成

```
prefectures       都道府県マスタ
municipalities    市区町村マスタ
jurisdiction_offices  管轄機関マスタ（市区町村に紐づく）
procedures        手続きマスタ
procedure_documents   手続き必要書類
official_links    公式リンク
companies         診断入力の保存（匿名）
```

## データ拡張方法

新しい都市を追加するには:
1. `municipalities` に市区町村を INSERT
2. `jurisdiction_offices` にその市区町村の管轄機関を INSERT

業種別手続きを追加するには:
1. `procedures` に `applicable_industries = ['restaurant']` などで INSERT
2. 対応する `procedure_documents`・`official_links` を INSERT
```
