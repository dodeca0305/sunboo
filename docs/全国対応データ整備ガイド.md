# 全国対応データ整備ガイド

SUNBOO経営ナビは現在MVPとして東京都渋谷区のみに対応しています。本ガイドは、対応エリアを他の市区町村・都道府県に拡張する際の作業手順をまとめたものです。

対象読者：管轄機関データ（税務署・年金事務所・労基署・ハローワーク・都税事務所・区市町村税窓口）を追加する運営担当者。

---

## 1. 全体像

```
supabase/import_templates/*.csv   ← ここを編集する（Excel/スプレッドシートでOK）
        │
        ▼  Supabase Table Editor で staging_* テーブルにインポート
staging_municipalities
staging_jurisdiction_offices
staging_official_links
        │
        ▼  supabase/import_from_csv.sql の STEP 2 を SQL Editor で実行
municipalities / jurisdiction_offices（本番テーブル）
```

CSVを直接本番テーブルにインポートしない理由：本番テーブルは `prefecture_id` / `municipality_id` のような外部キーを持っており、CSVの `pref_code` や `muni_code` のような「人間が読めるコード」とは形が違います。そのため、一度フラットな構造の staging テーブルに読み込んでから、SQLでJOINして本番テーブルへ変換・マージします。

---

## 2. データ追加手順

### Step 1: CSVテンプレートを編集する

`supabase/import_templates/` にある3つのCSVを開き、追加したい市区町村・機関の行を追記します。

| ファイル | 内容 | キー |
|---|---|---|
| `municipalities.csv` | 都道府県・市区町村 | `muni_code` |
| `jurisdiction_offices.csv` | 管轄機関の基本情報（名称・住所・電話・URL） | `muni_code` + `office_type` |
| `official_links.csv` | 管轄機関の公式リンク生存状況 | `muni_code` + `office_type` |

- `muni_code` は総務省の全国地方公共団体コード（5桁、例：渋谷区 = `13113`）を使用してください。
- `office_type` は以下の6種類のいずれかを使用してください（新しい種類を追加する場合はアプリ側の表示ロジックの対応も必要になるため、事前に開発担当に相談してください）。

| office_type | 内容 |
|---|---|
| `tax_office` | 税務署（国税） |
| `prefectural_tax` | 都道府県税事務所 |
| `municipal_tax` | 市区町村役所の税務窓口 |
| `pension_office` | 年金事務所 |
| `labor_standards` | 労働基準監督署 |
| `hello_work` | ハローワーク |

### Step 2: ステージングテーブルを作成する（初回のみ）

Supabase ダッシュボード → **SQL Editor** で `supabase/import_from_csv.sql` の **STEP 1** 部分を実行し、`staging_municipalities` / `staging_jurisdiction_offices` / `staging_official_links` の3テーブルを作成します。既に作成済みなら再実行しても問題ありません（`CREATE TABLE IF NOT EXISTS`）。

### Step 3: CSVをステージングテーブルにインポートする

Supabase ダッシュボード → **Table Editor** で、対象の `staging_*` テーブルを開き、右上の「Insert」→「Import data from spreadsheet」からCSVファイルを選択します。CSVのヘッダー名（1行目）がテーブルの列名と一致しているため、列は自動でマッピングされます。

> 2回目以降のインポートで前回分のデータを残したくない場合は、先に `import_from_csv.sql` の STEP 4 にある `TRUNCATE staging_xxx;` を実行してからインポートしてください。

### Step 4: 本番テーブルへマージする

SQL Editor で `supabase/import_from_csv.sql` の **STEP 2** を実行します。`ON CONFLICT DO UPDATE` で冪等化されているため、既存の市区町村・機関を含めて何度実行しても重複行は作られません。

- `municipalities.csv` を反映していない場合は 2-1 のブロックだけコメントアウトしてスキップできます。
- 新規の市区町村を追加する場合は、必ず `municipalities.csv`（2-1）→ `jurisdiction_offices.csv`（2-2）→ `official_links.csv`（2-3）の順に実行してください（`jurisdiction_offices` の行は `municipalities` に、`official_links.csv` の更新対象は `jurisdiction_offices` の行に依存するため）。

### Step 5: 反映結果を確認する

`import_from_csv.sql` の **STEP 3** の確認クエリで、追加した市区町村・機関が正しく登録されたか確認します。加えて、アプリ側でも実際の表示を確認してください（「6. 追加後の確認URL」を参照）。

## 3. 公式リンクの扱い

このアプリには「公式リンク」に関する2つの異なる概念があります。混同しないよう注意してください。

### (A) `jurisdiction_offices.official_url` 系カラム（本ガイドの対象）

各管轄機関（税務署・年金事務所など）自身のWebページが生きているかどうかを管理する列です。`office_type` × `municipality` ごとに1つ存在します。`official_links.csv` はこちらを更新します。

- `official_url`：機関の公式ページURL
- `official_url_status`：`ok` / `broken` / `redirected` / `unchecked`
- `official_url_checked_at`：最終確認日時
- `fallback_url`：リンク切れ時に案内する安定した公式一覧ページ

### (B) `official_links` テーブル（procedures に紐づく・全国共通）

「法人設立届出書」「青色申告承認申請書」など**手続きそのもの**の申請書式・案内ページへのリンクです。国税庁・厚労省・日本年金機構の全国共通ページを指しているため、**市区町村が変わっても変化しません**。新しい市区町村を追加しても、このテーブルにレコードを追加する必要は基本的にありません（`supabase/seed.sql` で既に10件投入済み）。

新しい都道府県で国税庁・年金機構以外の**独自の手続き**が必要になった場合のみ、`procedures` / `official_links` テーブルへの追加を検討してください（本CSV基盤の対象外・個別対応）。

## 4. fallback_url の考え方

`fallback_url` は「個別ページがリンク切れになったときに、ユーザーを迷子にしないための逃げ先」です。以下の方針で設定してください。

- **全国・都道府県単位で安定しているページ**（例：国税庁の税務署一覧、東京都主税局の都税事務所一覧、日本年金機構の年金事務所一覧、厚労省東京労働局の労基署一覧、東京ハローワーク一覧）は、同じ都道府県内であれば `office_type` ごとに使い回せます。
- **市区町村独自のページ**（`municipal_tax`＝区市町村役所の税務窓口）は、自治体ごとにURL構造が異なるため、使い回せる全国共通の fallback は存在しません。該当自治体の公式サイトのトップページや「税金」カテゴリページなど、担当者が個別に調査して設定してください。テンプレートCSVでは `municipal_tax` の `fallback_url` を空欄にしてあります。
- fallback_url が見つからない・未調査の場合は空欄のままで構いません。アプリ側は `fallback_url` が空でも動作します（リンク切れ表示のみ簡素になります）。

## 5. link_status の運用

`official_url_status` は以下の4値のいずれかです。

| 値 | 意味 | 設定タイミング |
|---|---|---|
| `unchecked` | 未確認（デフォルト） | 新規追加時、URLをまだ人力で開いて確認していない場合 |
| `ok` | 正常にアクセスできることを確認済み | 担当者が実際にリンクを開いて確認した場合 |
| `broken` | リンク切れ（404等）を確認済み | 定期リンクチェックで検出した場合 |
| `redirected` | 別URLへリダイレクトされる（要更新） | 定期リンクチェックで検出した場合 |

**運用ルール：**

- 新規にCSVでデータを投入する際は、URLを実際に開いて確認できたもののみ `ok` としてください。未確認のまま `ok` にしない（本ガイドのサンプルCSVでは新宿区・港区の行を意図的に `unchecked` にしてあります）。
- `broken` / `redirected` を検出した場合は、正しいURLに更新した上で `official_url_status` を `ok` に戻し、`fallback_url` があわせて古くなっていないか確認してください。
- 将来的にリンクチェックを自動化する場合（GitHub Actions 週次など）も、更新先はこの `official_url_status` / `official_url_checked_at` 列を想定しています。

## 6. 追加後の確認URL

新しい市区町村を追加したら、`muni_code` を使って以下のURLで表示を確認してください（`pref` は都道府県コード、`muni` は市区町村コード、`emp` は従業員の有無、`fm` は決算月）。

```
# 例: 新宿区（東京都・muni_code=13104）・従業員あり・決算月3月
http://localhost:3000/result?pref=13&muni=13104&emp=true&fm=3

# 例: 港区（東京都・muni_code=13103）・従業員あり・決算月3月
http://localhost:3000/result?pref=13&muni=13103&emp=true&fm=3
```

確認ポイント：

| 確認項目 | 期待する状態 |
|---|---|
| `/start` の市区町村セレクト | 追加した市区町村が選択肢に表示される |
| `/result` の管轄機関 | 追加した6機関（またはCSVで投入した分）が表示される |
| `/offices` | 一覧に追加した機関が表示され、タイプフィルターが動作する |
| 公式リンクの表示 | `official_url_status` が `unchecked` の機関は「（未確認）」ラベルが表示される |

本番（Vercel）に反映する場合は、`https://your-app.vercel.app` に置き換えて同様に確認してください。
