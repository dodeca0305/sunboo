## 3. 5分テストの進め方

対象: 士業・多忙な相手・第一印象だけ見たい相手。

1. トップページ（`/`）を開いてもらい、**何も説明せず**「これは何をするサービスだと思いますか」と聞く
   → [user-test-feedback.md](user-test-feedback.md) Q1
2. 「デモとして試す」ボタンを押してもらい、`/result`へ進んでもらう（フォーム入力は発生しない）
3. 画面を1分ほど自由に眺めてもらう。特に「通知」「AI参謀」カードに自分から気づくか観察する
4. 気づかなければこちらから「上の方に2つカードがありますが、それぞれ何をしていると思いますか」と聞く
   → Q3（AI参謀の理解度）
5. 「これは役に立ちそうですか」「顧問先に薦めたいと思いますか」を聞く → Q4, Q5, Q8, Q8-a
6. 「月額いくらなら払えると思いますか」を聞く → Q9
7. 実施記録欄・回答をその場で記入し終える（後回しにしない）

所要目安: 説明1分＋操作2分＋ヒアリング2分。

## 4. 15分テストの進め方

対象: 起業予定者・起業1年以内の経営者、じっくり見てもらいたい相手。

1. トップページを開いてもらい、5分版と同様に第一印象を聞く → Q1
2. 「はじめる」から`/start`へ進み、実際に自社（または想定の会社）の情報を入力してもらう
   → 入力に迷った項目をその場でメモ（Q2）
3. `/result`の管轄機関一覧・必要手続き一覧を眺めてもらい、情報量・レイアウトの感想を聞く → Q6
4. 「通知」「AI参謀」カードについて5分版と同様に確認する → Q3, Q4, Q5
5. 必要手続きを1件選び「詳細を見る」を開いてもらう。必要書類・注意書きが役立ちそうか確認する
6. 任意の手続きのステータスボタンを押してもらい、完了にした手続きがAI参謀・通知から
   リアルタイムに外れることに気づくか観察する
7. `/events`から任意のイベント（例: 会社設立）を登録してもらい、手続きが自動生成される様子を見てもらう
8. フィードバックシートの残り全項目（Q7〜Q11、その他自由記述）に回答してもらう

所要目安: 説明1分＋入力2分＋結果確認5分＋詳細/ステータス操作3分＋イベント登録2分＋ヒアリング2分。

## 5. フィードバック集計方法

1. 個票（[user-test-feedback.md](user-test-feedback.md)のコピー）が5名分そろっていることを確認する
2. [user-test-summary.md](user-test-summary.md)を開き、以下の順で転記する
   - 「1. 参加者属性」: 各個票の属性にチェックを立てて人数を数える
   - 「2. 診断完了率」「3. イベント登録率」: 各個票の「実施記録」欄から集計（自己申告ではなく観察事実を使う）
   - 「4. AI参謀の理解度」「5. 通知カードの有用性」: Q3・Q5の選択肢ごとに人数を数える
   - 「6. 分かりにくかった箇所」: Q7の自由記述を読み、同じ趣旨のものをグルーピングして件数を記入
   - 「7. 支払意思」: Q9の金額を並べて最小・最大・中央値・平均を算出
3. 「8. 改善優先度」「9. 総合所感」は集計後、次項の判断会で埋める（テスター個人の意見ではなくチームの決定）

## 6. テスト後に次Sprintを決める判断基準

集計結果を見ながら、**上から順に**該当する条件を確認する。複数該当する場合は、より上にある基準を優先する
（土台が崩れている問題を、表面的な改善より先に直すため）。

| 優先順位 | 条件 | 次Sprintの方向性 |
|---|---|---|
| 1 | 診断完了率が80%未満、または同じ箇所での離脱が2名以上 | 新機能追加を止め、離脱箇所の入力UX修正を最優先のSprintにする |
| 2 | AI参謀の理解度が「ほとんど理解できなかった」が過半数 | AI参謀のUI/コピーの分かりやすさ改善に専念するSprintにする（v0.8.1の延長） |
| 3 | 通知カードの有用性が「不要」が過半数 | 通知カードは追加開発せず現状維持、または統合・簡略化を検討するSprintにする |
| 4 | 「分かりにくかった箇所」が特定の1〜2画面に集中している | その画面のUXピンポイント改善をSprintのテーマにする |
| 5 | 支払意思が0円回答が過半数、または全体的に著しく低い | 機能改善ではなく価値提案・料金モデルの見直しをチームで議論する（Sprintの前に経営判断が必要） |
| 6 | 上記いずれにも該当せず、実務で使いたい／顧問先に案内したいが過半数で好意的 | 大きな課題は無いと判断し、`docs/ROADMAP.md`のv1.0正式リリース準備（対応エリア拡大・料金設計等）に進む |

判断会では、該当した優先順位の行を`docs/user-test-summary.md`の「8. 改善優先度」「9. 総合所感」に転記し、
次Sprintの依頼文（ユーザーからの次回の指示）にそのままつながる形で1行サマリを残す。
