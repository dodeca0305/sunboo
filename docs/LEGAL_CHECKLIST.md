## Version 1.0まで残る法務課題

1. **運営者の法的地位が未確定。** 個人事業か法人か、正式名称・所在地が定まっていないため、
   利用規約第1条・第8条、プライバシーポリシーの事業者情報欄が確定できない。エンジニアリング
   作業ではなく、運営者自身の意思決定が必要
2. **弁護士による正式レビューが未実施。** 本書・両ドラフトはいずれも一般的なひな形の
   組み合わせであり、SUNBOOの実際の事業内容（行政手続き情報サービス、顧問先の会社情報を
   預かるBtoBtoC構造）に対する法的な適合性確認ができていない
3. **問い合わせ窓口が個人アドレスのまま。** [docs/LAUNCH_OPERATIONS.md](LAUNCH_OPERATIONS.md)・
   [docs/V1_READINESS_CHECKLIST.md](V1_READINESS_CHECKLIST.md)で既出の課題と同一
4. **開示等請求・Cookie同意の実務フローが未設計。** ポリシー上「請求を受け付ける」と
   書いても、実際に受け取ってから対応する手順（誰が・どの期間で・どう本人確認するか）が
   存在しない

## 弁護士レビューが必要な項目（優先順）

1. **利用規約第5条（免責事項）・第8条（準拠法・裁判管轄）** — 運営形態確定後、最優先で
   専門家確認が必要。効力の有無に直結する
2. **プライバシーポリシー5節（第三者提供）・7節（開示等請求への対応）** — 個人情報保護法上の
   該当性判断・実務フローの適法性確認が必要
3. **Cookie方針の同意バナー要否** — 現状「同意バナーなし」で運用する整理の最終確認
4. **利用規約・プライバシーポリシー全体の業際規制適合性** — 税理士法・行政書士法等、
   SUNBOOが「情報を見る／自動生成する」の範囲を超えていないかの確認
   （[CLAUDE.md](../CLAUDE.md)が定める製品範囲そのものの法的な裏付け）
