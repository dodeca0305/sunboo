## 0. 手法・スコープ

- **公開画面（Top/Start/Profile/Events/Result）**：`playwright-core`（npm依存追加なし、キャッシュ済みChromium使用）で
  375px/768px/Desktop(1280px)の実機スクリーンショットを取得し、目視確認した。Resultはローカルストレージへ
  実在の会社プロフィールを直接投入し、実データ（Supabase seed）で計算された診断結果を確認した。
  加えて、キーボードのTab操作・focus-visible・見出し階層・アイコンのみボタンのaria-label・
  実際に描画された色のコントラスト比を、ブラウザの`getComputedStyle`から直接取得して検証した。
- **管理画面（Dashboard/Roadmap/Profile/Documents/Share）**：本セッションには管理者ログイン情報が無く
  （`README.md`のセットアップ手順が示す通り、管理者アカウントはSupabase Authへの手動登録が必要で、
  リポジトリに認証情報は含まれていない）、実機確認はできなかった。**Sprint82〜85で実装した内容の
  コードレビューのみ**に基づく評価とし、その旨を各所に明記する。
- **文言監査**：`当該`・`該当する場合`・`処理に失敗しました`・`エラーが発生しました`・`入力してください`の
  禁止語・要注意語を`src/`全体でgrep検索した。
- **印刷**：`@media print`ルール・`print:`Tailwindバリアントの有無をコードベース全体で検索した。
- **コントラスト**：実際にブラウザが計算した色（`rgb(...)`）からWCAG相対輝度式で比率を算出した
  （SUNBOO Tokenは過去スプリントで検証済みの値を再利用し、本スプリントでは主に「まだToken化されていない
  既存Tailwind色」を対象に計算した）。

## 3. High（4件）

### H-1. Body背景が依然として白（`#ffffff`）で、WarmPaper（`#FAF9F6`）が一度も本番反映されていない
`src/app/globals.css:17-20`（`@layer base { body { background-color: #ffffff; ... } }`）。Sprint82の
Shared UI Migration時点で「対象6クラスに含まれないため変更していない」と明記された既知の未着手項目が、
Sprint83〜85でも手を付けられないまま残っている。SUNBOOブランドの中核トークンであるWarmPaperが
**アプリのどのページでも一度も画面に反映されていない**状態であり、「紙の手帳感」（Sprint85 Phase4の
狙い）の実現を土台から妨げている。
- **影響範囲**：全ページ（公開・管理画面問わず）
- **推奨対応**：`body`の`background-color`をWarmPaperに変更する回帰テスト付きの小さな専用スプリントを
  Sprint87の最優先候補にする

### H-2. 公開画面の主要導線にBlue-600系の色が広範に残存（56箇所）
`grep`で`src/app/(site)/**/*.tsx`・`src/components/*.tsx`を検索した結果、19ファイル・56箇所で
`blue-*`系Tailwindクラスが現存する。特に集中しているのは診断結果画面本体：

| ファイル | 概要 |
|---|---|
| `src/app/(site)/result/ScheduleList.tsx` | AI参謀カード（`border-blue-100 bg-blue-50/40`）、最重要アクションボックス、スター評価（`fill-blue-600`）、手続き完了率プログレスバー（`bg-blue-600`）など19箇所 |
| `src/app/(site)/result/page.tsx` | 管轄機関カードのアイコン等7箇所 |
| `src/app/(site)/start/page.tsx` | 手順番号バッジ（①〜⑤、`bg-blue-600`）5箇所 |
| `src/app/(site)/roadmap/page.tsx`（公開） | 見出しアイコン・β版タグ3箇所 |
| その他（`profile`・`events`・`offices`・`procedures`・`profile/tax-returns`等） | 選択状態・アイコン・確認バナー等 |

Sprint82ではDesign Tokensを共通クラス（`.card`等）にのみ接続し、Sprint83では「Procedure Status」に
限定してScheduleListの一部（StatusButton・進行中タグ）のみ再配色した。**「AI参謀」カード自体や
スター評価、進捗バーはスコープ外として意図的に据え置かれたまま**であり、初めて診断結果を見る
ユーザーが最初に目にする「Result」画面の主要な視覚要素が、他のSprintで刷新されたMorningSun/Ink
パレットと混在している。
- **影響範囲**：`/result`（診断の最終到達点、最も重要な画面の1つ）が中心
- **推奨対応**：Sprint83のProcedure Status統一の延長として、AI参謀カード・スター評価・進捗バーを
  対象にした専用スプリントを設ける

### H-3. `text-gray-400`のコントラスト比がWCAG AA基準未達（約2.54:1、必要基準4.5:1）
`text-gray-400`（`#9CA3AF`）を実際にブラウザで白背景上に描画させ、相対輝度からコントラスト比を
算出したところ**約2.54:1**だった。WCAG AA（通常テキスト4.5:1）は元より、大きな文字/UIコンポーネント
向けの3:1基準すら下回る。この値は、SUNBOO Tokenの「Ink Muted」（`#6B7280`、コントラスト比約4.83:1、
AA適合）とは別の、旧来のTailwindグレーであり、`grep`によれば**19ファイル・77箇所**で現存する
（例：`(site)/procedures/ProcedureList.tsx`の「（未確認）」注記、各所のタイムスタンプ・キャプション類）。
- **影響範囲**：公開画面全体に散在する補足テキスト・キャプション
- **推奨対応**：`text-gray-400`を`text-sunboo-ink-muted`へ一括置換する（値の意味用途はほぼ同一のため
  低リスクな置換だが、置換後の視認性を目視確認するステップは必要）

### H-4. ブラウザの直接印刷（`Cmd+P`）に対応した`@media print`が存在しない
`src/`全体を検索したが、`@media print`ルール・Tailwindの`print:`バリアントは**1件も存在しない**。
Sprint84でRoadmap Card自体には`break-inside-avoid`を付与したが、それ以外（ヘッダーナビ・フッター・
「詳細を見る」ボタン・Excel/PDFエクスポートボタン等の操作用UI）を印刷時に非表示にする指定が無いため、
利用者がRoadmapページを直接`Cmd+P`で印刷すると、ナビゲーション・ボタン等の非本質要素も
そのまま紙に出力される。PDF出力ボタン（`roadmapPdfDocument.ts`、Sprint84で検証済み・印刷品質は良好）
という代替が既に用意されているが、「ブラウザ印刷」という確認項目自体には対応していない。
- **影響範囲**：`(site)/roadmap`・`admin workspaces/[id]/roadmap`・`share/[token]`
- **推奨対応**：最小限の`@media print { header, footer, button { display: none } }`をSprint87で追加する
  （PDF出力を持たない`share/[token]`では特に効果が大きい）

## 4. Medium（6件）

### M-1. `/result`の診断成功時ビューに`<h1>`が存在しない
`src/app/(site)/result/page.tsx`は、入力不足時のエラービュー（69行目）にのみ`<h1>`があり、
**実際の診断結果（成功時）のビューには`<h1>`が無く**、いきなり`<h2>`（「管轄機関」「必要手続き」）から
始まる。スクリーンリーダー利用者がページの主題を見出しレベルから把握できない。

### M-2. 禁止語「エラーが発生しました」が2箇所残存
`src/app/admin/(protected)/workspaces/error.tsx:28`、`src/app/(site)/roadmap/page.tsx:76`。
Sprint80のガイドライン策定時から指摘済みだが、Sprint85で管理画面Roadmapのエラー文言は
改善した一方（「ロードマップを計算できませんでした」）、公開`/roadmap`側とWorkspaceのエラー境界
（`error.tsx`）は据え置きのままだった。

### M-3. 「〇〇を入力してください」の単独命令形が9箇所残存
Sprint83監査から状態変化なし。フィールド名を伴っており意味は明確だが、ガイドライン上は
「入力内容を確認してください」等への言い換えが推奨されている。

### M-4. Workspace Profile／公開Profileとも情報量が多く「読み疲れ」のリスク
Workspace Profileは7つのカードに多数のSegmentedControlが並び、公開Profileも同様の構成。
Sprint85のBrand Auditで既に指摘済みで、今回のレビューでも改善は確認できなかった
（フォーム項目数そのものを減らす設計変更はスコープ外のため、今回も未着手のまま）。

### M-5. 一部のインタラクティブ要素がブラウザ既定のfocus outlineのまま
`WorkspaceSubNav.tsx`のタブリンク、`ScheduleList.tsx`の`StatusButton`は、SUNBOO共通の
`.btn-secondary`/`.segmented-option`（Ink・2px・角丸に沿った専用outline）を使わず、素の
`<a>`/`<button>`のためブラウザ既定（`outline: auto`）に依存している。機能的にはfocus-visibleが
効いており「見えない」わけではないが、他の要素と視覚的に統一されていない。

### M-6. 通知（「確認が必要なこと」）の位置づけが「今日のポイント」と一部重複
Sprint85のBrand Auditで既出。Dashboard内で2番目のカードとして残っており、最上部の
「今日のポイント」との役割分担が今回のレビューでも明確に解消されていない。

## 5. Low（3件）

### L-1. ページレベルとアイテムレベルの「情報不足・推定」説明が粒度違いで重複気味
Roadmapページ冒頭の一般的な注記と、Sprint85で追加したカード内の個別説明文が、粒度は異なるが
似た内容を繰り返す。実害は小さいが、将来的に一本化の余地がある。

### L-2. Start/Eventsの手順番号バッジ（①〜⑤）がBlue-600のまま
H-2と同根だが、装飾的な数字バッジのみのため実害は小さく個別にLowとして記録する。

### L-3. `WorkspaceShareLinksPanel`等、UI操作は問題ないが軽微な文言の硬さが残る
「発行に失敗しました」等、フィールドの文脈上意味は明確だが、禁止語リストには入っていないものの
やや事務的な言い回しが残る箇所が散見される。

## 7. 開始前に直すべきもの（Sprint87最優先候補）

1. **H-3（コントラスト未達）**：`text-gray-400`→`text-sunboo-ink-muted`の一括置換。実害が大きく、
   修正リスクが低いため最優先
2. **M-1（h1欠落）**：`/result`成功時ビューへの`<h1>`追加。影響範囲が1ファイルに限定され対応が軽い

上記2件は影響範囲が明確でリスクが低いため、Closed Beta開始前でも短時間で対応可能と判断する。

## 8. 開始後で良いもの

- H-1（WarmPaper未適用）・H-2（Blue-600残存）・H-4（印刷未対応）：いずれも「ブランド体験の完成度」の
  問題であり、Closed Betaの目的（機能的なフィードバック収集）を妨げない。まとまった作業量が
  必要なため、Sprint87以降で計画的に着手することを推奨する
- Medium/Low全件：機能に影響しない、または影響範囲が限定的なため、Closed Beta中のフィードバックと
  合わせて優先順位を再検討してよい
